# 핵심 갭: 검증 루프 부재 + 실무 괴리

## 문제 (원인)

### 1. 검증 메커니즘 전무
학습자가 코드를 작성한 후 **"이게 맞는가?"를 판단할 방법이 없음**.

현재:
- Sandpack: 에러(SyntaxError, TypeError)만 보임
- 자동 테스트: 없음
- 정답 코드: 없음
- 피드백: 없음

결과: **여전히 "튜토리얼 따라하기" 문제 미해결** ("맞나?" 불명확)

### 2. 실무 적응성 급격히 낮음
배운 내용의 **절반 이상을 실무에서 직접 쓸 수 없음**.

수치:
- useState: 53% 직접 적용 (나머지는 라이브러리나 다른 방법)
- useEffect: 40% 직접 적용 (TanStack Query 필수)
- Custom Hooks: 라이브러리가 이미 제공
- Context: 실무는 Zustand 사용
- Focus Trap, useImperativeHandle: 라이브러리가 제공

결과: **49개 관문을 다 배워도, 실무는 완전히 다른 스택**

### 3. 라이브러리 생태계 미포함
49개 관문 이후 실무를 시작하면:

```
학습자가 배운 것:
  ├─ useState (상태 관리)
  ├─ useEffect (데이터 fetch)
  └─ Custom Hooks (로직 재사용)

실무에서 필요한 것:
  ├─ TanStack Query (데이터 fetch의 정답)
  ├─ Zustand (전역 상태 관리)
  ├─ React Hook Form (폼 처리)
  ├─ Next.js App Router 심화
  └─ 성능 측정 도구
```

결과: **"배운 것" 대비 **"배우지 않은 것"이 더 중요**

## 임팩트

### 학습자 심리
```
1단계: "49개 관문을 다 배웠다! 이제 실무 갈 수 있겠다"
2단계: 프로젝트 시작 → "어? TanStack Query는?"
3단계: "이건 배우지 않았는데?"
4단계: "내가 배운 게 왜 바로 안 써?"
5단계: 😭 (불안감, 자신감 하락)
```

### 프로젝트 목표와의 괴리
```
목표: "React 핵심 원리를 체득하고 실무에 적용"
현실: "React 핵심 원리만 배움, 실무는 완전히 다름"
```

## 우선 해결책 (v0.3)

### P0-1: 검증 시스템
1. **정답 코드**: 모든 관문에 solutionCode 추가
2. **테스트 케이스**: 자동 검증 (통과/실패)
3. **피드백**: 사용자 코드 vs 정답 비교

### P0-2: 진도 추적
1. **localStorage**: 완료 상태 저장
2. **UI**: 배지, 진도율, 마지막 관문 추적

### P0-3: 힌트 시스템
1. **단계적 도움**: Hint 1 → Hint 2 → 정답
2. **사용자 선택**: "다음 힌트" vs "정답 보기"
