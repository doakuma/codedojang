# 코드도보통지 — UI/UX 점검 보고서

> 작성일: 2026.03.27
> 범위: v0.2 현재 상태 (홈, 수련장, 문파, 관문 페이지)
> 우선순위: 높음(H), 중간(M), 낮음(L)

---

## 1. 네비게이션 및 사용자 위치 파악

### ✅ 현재 상태
- 관문 페이지에 브레드크럼 네비게이션 구현됨
- 헤더에 로고 + "수련장" 링크로 홈 복귀 가능
- 관문 이전/다음 링크로 선형 이동 가능

### ⚠️ 개선사항

| 항목 | 문제 | 개선안 | 우선순위 |
|------|------|--------|---------|
| 관문 네비게이션 크기 | `text-sm`으로 너무 작음 (특히 모바일) | `text-sm md:text-base`로 확대 | M |
| 이전/다음 버튼 모바일 표시 | 한 줄 배치일 때 "목록으로" 버튼 밀림 | flex-col md:flex-row로 변경 또는 justify-center 적용 | M |
| 관문 완료 후 위치 표시 | v0.2.x 진도 추적 시 "현재 관문" 표시 필요 | 별도 시각 표시 (스타 아이콘 등) 추가 | M |

## 2. 시각적 계층 및 일관성

### ✅ 현재 상태
- 문파 카드 레이아웃 일관됨 (홈 = 수련장 페이지)
- 관문 카드 일관된 디자인 (모든 문파에서 동일)
- Tailwind 디자인 토큰 일관되게 사용 (bg-card, text-foreground 등)

### ⚠️ 개선사항

| 항목 | 문제 | 개선안 | 우선순위 |
|------|------|--------|---------|
| 섹션 헤더 가시성 | GatePage의 "📖 이론 비급" / "⚔️ 수련 코드"가 `text-xs` (너무 작음) | `text-sm font-semibold`로 확대 | M |
| 문파 페이지 Grid 갭 | 홈/수련장: gap-6, 관문 목록: gap-4 (불일치) | 모두 gap-6으로 통일 | L |
| 홈 페이지 스페이싱 | 다른 페이지는 `container max-w-5xl pt-8 pb-10`, 홈은 `p-8` | 일관성 있게 구조 정리 | L |

## 3. 진도 추적 시각성 (v0.2.x 예정)

### 📋 현재 지원 상태
- `SooryeonCard`의 `isApproved` prop으로 완료 배지 표시 가능 (초록색)
- 문파 페이지에 완료율 표시 UI 구현 예정

### ⚠️ 개선사항

| 항목 | 문제 | 개선안 | 우선순위 |
|------|------|--------|---------|
| 색상만으로 상태 표시 | 초록색 배경만으로는 색맹 사용자 구분 어려움 | 텍스트 레이블 ("완료") + 체크마크 아이콘 추가 | H |
| 문파 완료율 표시 방식 | 완료한 관문 수 / 전체만으로는 진도 직관성 부족 | 프로그레스 바 시각화 추가 | M |
| 관문 카드 hover 상태 | 완료 여부가 정적으로만 표시됨 | 마우스 hover 시 "완료됨" 배지/텍스트 강조 | M |

## 4. 코드 에디터 (CodeEditor / Sandpack)

### ✅ 현재 기능
- 실시간 React 코드 실행 (Sandpack iframe 격리)
- 콘솔 출력 및 인라인 에러 표시
- 라인 번호 표시
- 리사이징 가능 패널
- 의존성 지원 (immer, zustand, @tanstack/react-virtual)

### ⚠️ 개선사항

| 항목 | 문제 | 개선안 | 우선순위 |
|------|------|--------|---------|
| 코드 초기화 버튼 없음 | 사용자가 코드를 망가뜨렸을 때 되돌릴 방법 없음 | "초기 코드 복원" 버튼 추가 (마우스 아이콘 클릭) | H |
| 어두운 테마만 지원 | `theme="dark"` 고정 | 라이트/다크 토글 추가 또는 시스템 설정 반영 | M |
| 높이 고정 (480px) | 모바일에서 너무 크거나 짧을 수 있음 | 반응형 높이: 모바일 320px, 데스크톱 480px | M |
| 코드 저장 불가 | 작성한 코드가 페이지 새로고침 시 사라짐 | localStorage에 draft 저장 (v0.3 미래) | L |
| 해결책 코드와 에디터 연결 안 됨 | 모달 내 해설 코드를 직접 에디터로 복사 불가 | "이 코드 실행해보기" 버튼 추가 (v0.3) | M |

## 5. 모바일 반응형 디자인

### ✅ 현재 상태
- `grid-cols-1 md:grid-cols-2 lg:grid-cols-3`으로 기본 반응형 구현됨
- Header sticky 레이아웃 모바일 지원

### ⚠️ 개선사항

| 항목 | 문제 | 개선안 | 우선순위 |
|------|------|--------|---------|
| Sandpack 에디터 높이 | 480px 고정은 모바일에서 스크롤 부담 | 모바일 시 높이 제약 또는 접기 옵션 | M |
| 관문 이전/다음 네비게이션 | `flex items-center justify-between`은 작은 화면에서 버튼끼리 부딪힐 수 있음 | flex-col md:flex-row 또는 더 작은 텍스트 | M |
| 섹션 패딩 | `pt-8 pb-10 space-y-8`이 모바일에서 과도할 수 있음 | 반응형: `pt-4 md:pt-8` 등 | L |

## 6. 텍스트 및 가독성

### ✅ 현재 상태
- 제목 크기 계층이 명확함 (h1 text-3xl, h2 text-xl)
- 색상 대비 좋음 (foreground vs muted-foreground)

### ⚠️ 개선사항

| 항목 | 문제 | 개선안 | 우선순위 |
|------|------|--------|---------|
| 관문 미션 텍스트 | `text-lg text-muted-foreground`로 강조 부족 | `text-muted-foreground` 제거 또는 `text-foreground`로 변경 | M |
| 섹션 헤더 | "📖 이론 비급" / "⚔️ 수련 코드"가 `text-xs`로 너무 작음 | `text-sm font-semibold md:text-base` | M |
| 마크다운 렌더링 줄 높이 | 마크다운 콘텐츠의 줄 간격이 기본값일 수 있음 | `leading-relaxed` 또는 `leading-7` 적용 | L |

## 7. 비어있는 상태 처리 (Empty States)

### ⚠️ 현재 부족한 부분

|---------|------|--------|---------|

| 잘못된 URL | `notFound()` 처리됨 | ✅ 현재 상태 양호 | — |

## 8. 접근성 (Accessibility)

### ✅ 현재 지원
- `<sr-only>` (GitHub 링크)
- `title` 속성 (비급 보기 아이콘)

### ⚠️ 개선사항

| 항목 | 문제 | 개선안 | 우선순위 |
|------|------|--------|---------|
| 아이콘 aria-label | 일부 아이콘(ChevronRight 등)에 aria-label 없음 | 모든 아이콘에 `aria-label` 또는 `title` 추가 | M |
| 색상 대비 (다크모드) | 완료 배지의 초록색 텍스트가 어두운 배경에서 대비 약할 수 있음 | WCAG AA 기준 확인 (4.5:1) | H |
| 폼 요소 라벨 | CodeEditor에 사용 안내 텍스트 없음 | 에디터 위에 "아래 코드를 수정하고 결과를 확인하세요" 안내 | M |
| 키보드 네비게이션 | 전반적으로 tabindex 명시 없음 | 탭 순서 확인 및 필요시 조정 | L |

## 9. 상호작용 피드백

### ✅ 현재 상태
- 카드 hover: `hover:shadow-lg transition-all hover:border-foreground/30`
- 링크 hover: `hover:text-foreground transition-colors`
- 버튼 클릭: 기본 shadcn/ui 상태

### ⚠️ 개선사항

| 항목 | 문제 | 개선안 | 우선순위 |
|------|------|--------|---------|
| 관문 완료 버튼 | 없음 (v0.2.x에서 추가 예정) | 클릭 시 확인 음성 피드백 또는 토스트 알림 추가 | M |
| Sandpack 테마 | 다크테마만 지원 (라이트 모드 미지원) | 시스템/사용자 선택 반영 | M |
| 로딩 상태 | Sandpack 초기화 시 로딩 상태 표시 없음 | 스켈레톤 또는 로딩 인디케이터 추가 | L |

## 11. 우선순위별 액션 아이템

### 🔴 높음 (High) — 즉시 실행 권장

1. **Sandpack 코드 초기화 버튼** — 사용성 필수
2. **색상 외 상태 표시** — 접근성 (WCAG AA)
3. **섹션 헤더 가시성** — 정보 구조 개선

### 🟠 중간 (Medium) — v0.3 계획에 포함

1. 모바일 반응형 (Sandpack 높이, 네비게이션)
2. 관문 미션 텍스트 강조
3. 관문 네비게이션 버튼 크기
4. 아이콘 aria-label
5. Sandpack 라이트/다크 토글
6. 모바일 수직 배치 검토

### 🟡 낮음 (Low) — 장기 개선

1. Grid gap 일관성 (gap-4 → gap-6)
2. 코드 저장 기능 (localStorage draft)
3. 마크다운 줄 높이 조정
4. 이론 콘텐츠 없음 상태 메시지
