## P0 — 앱 파손 (즉시 수정 필요)

### [P0-1] 홈 → 수련 페이지 링크 404
- **위치:** `src/app/page.js`, `src/data/roadmap.js`
- **원인:** `roadmap.js`의 토픽 ID가 `curriculum.js`의 실제 ID와 불일치.

  | roadmap.js (잘못된 ID) | curriculum.js (실제 ID) |
  |---|---|
  | `04-custom-hooks` | `04-customHooks` |
  | `06-performance` | `06-optimization` |
  | `07-nextjs` | `07-nextjs-rsc` |

- **결과:** 홈에서 4, 6, 7번 카드 클릭 시 404.
- **수정안:** `src/app/page.js`가 `roadmap.js` 대신 `curriculum.js`의 `getAllTopicIds()` + `getTopic()`을 사용하도록 변경하여 데이터 소스를 일원화. `roadmap.js`는 제거.

## P1 — 기능 결함

### [P1-1] `/training` 경로 404
- **위치:** `src/components/shared/GatePage.jsx` 브레드크럼
- **원인:** `GatePage` 브레드크럼의 "수련장" 링크가 `/training`을 가리키나 해당 라우트(`src/app/training/page.js`)가 존재하지 않음.
- **결과:** 관문 페이지에서 "수련장" 클릭 시 404.
- **수정안:** `src/app/training/page.js` 생성. 전체 문파 목록을 보여주는 인덱스 페이지.

### [P1-2] 홈 페이지가 `roadmap.js`에 의존 (데이터 이중화)
- **위치:** `src/app/page.js`, `src/data/roadmap.js`
- **원인:** 홈은 `roadmap.js`를, 수련 페이지는 `curriculum.js`를 각각 참조. 동일한 토픽 데이터가 두 파일에 분산.
- **결과:** 데이터 변경 시 두 파일을 동시에 수정해야 하며 불일치 발생.
- **수정안:** 홈 페이지도 `curriculum.js`에서 데이터를 읽도록 통일. `roadmap.js` 삭제.

## P2 — 디자인 불일치

### [P2-1] 홈 페이지 디자인 시스템 이탈
- **위치:** `src/app/page.js`
- **원인:** 홈은 `bg-zinc-50`, `text-zinc-900`, `bg-white` 등 Zinc 팔레트를 직접 사용. 수련 페이지는 `bg-background`, `text-foreground` CSS 변수 사용.
- **결과:** 홈과 내부 페이지 간 색상 체계가 다름. 다크모드에서 특히 두드러짐.
- **수정안:** 홈도 shadcn/ui 디자인 토큰(`bg-background`, `text-foreground`, `border-border` 등)을 사용하도록 통일.

### [P2-2] `lang="en"` — 한국어 앱에 영어 lang 속성
- **위치:** `src/app/layout.js`
- **수정안:** `lang="ko"` 로 변경.

### [P2-3] 메타데이터 기본값 미변경
- **위치:** `src/app/layout.js`
- **현재값:** `title: "Create Next App"`, `description: "Generated by create next app"`
- **수정안:** 앱 실제 정보로 업데이트.
  ```js
  export const metadata = {
    title: '코드도장 | React 정공법 수련',
    description: '7대 문파 49관문으로 체득하는 리액트 정공법',
  };
  ```

### [P2-4] 페이지 상단 여백 없음
- **위치:** `src/components/shared/SooryeonLayout.jsx`, `src/components/shared/GatePage.jsx`
- **원인:** 두 컴포넌트 모두 `pt-*` 클래스 없이 시작. 브레드크럼/제목이 뷰포트 최상단에 바로 붙음.
- **수정안:** 최상위 `div`에 `pt-8` 또는 `pt-10` 추가.

## P3 — 코드 품질

### [P3-1] `MarkdownViewer` — code block 미구분
- **위치:** `src/components/shared/MarkdownViewer.jsx`
- **원인:** `code` 컴포넌트가 인라인 코드(`` `foo` ``)와 코드 블록(` ```js ... ``` `)을 동일하게 인라인 스타일로 렌더.
- **결과:** 이론 비급의 코드 블록이 인라인 배지처럼 표시됨.
- **수정안:** `inline` prop으로 분기 처리.
  ```jsx
  code: ({ inline, ...props }) =>
    inline
      ? <code className="rounded bg-muted px-1.5 py-0.5 font-mono text-[0.8rem] text-primary" {...props} />
      : <pre className="rounded-lg bg-muted p-4 overflow-x-auto"><code className="font-mono text-sm" {...props} /></pre>
  ```

### [P3-2] `SooryeonCard` — 빈 CardContent / CardFooter 렌더
- **위치:** `src/app/training/[topic]/page.js`에서 `SooryeonCard` 사용 부분
- **원인:** `children`, `guideText`, `actionButton`, `solution`을 전달하지 않아도 `<CardContent>`, `<CardFooter>` 엘리먼트는 항상 렌더됨.
- **결과:** 카드 하단에 불필요한 빈 공간 발생.
- **수정안:** `SooryeonCard` 내부에서 해당 섹션을 조건부 렌더링으로 변경.

### [P3-3] `SooryeonLayout` — `onReset` 데드 코드
- **위치:** `src/components/shared/SooryeonLayout.jsx`
- **원인:** `onReset` prop이 정의되어 있으나 RSC에서 함수를 prop으로 전달할 수 없어 실제로 사용되는 곳이 없음. `Button`, `RotateCcw` 임포트도 미사용.
- **수정안:** 사용 계획이 없으면 제거. 진도 추적 기능 구현 시 재설계.

### [P3-4] `GatePage` — 취약한 문자열 파싱으로 토픽명 추출
- **위치:** `src/components/shared/GatePage.jsx`
- **현재:** `topic?.title?.split('. ')[1]?.split(':')[0]`
- **원인:** 제목 포맷(`"01. useState: 상태의..."`)에 강하게 결합됨. 포맷 변경 시 조용히 깨짐.
- **수정안:** `curriculum.js`의 `TOPICS`에 `shortTitle` 필드를 추가하거나 브레드크럼용 이름을 별도로 관리.
