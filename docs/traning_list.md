#### 1. useState: 불변성과 비동기 업데이트의 이해

상태 변경 시 가장 기초적이면서도 가장 많이 실수하는 케이스들입니다.

- **객체/배열 직접 수정:** `state.push()`처럼 원본을 건드려 렌더링이 안 되는 경우 (불변성 유지 수련)
- **이전 상태 의존성:** `setCount(count + 1)`을 연달아 호출할 때 값이 씹히는 현상 (함수형 업데이트 사용)
- **비동기 스냅샷:** `setState` 직후에 `console.log(state)`를 찍으면 이전 값이 나오는 이유 이해하기
- **과도한 상태 분화:** 서로 연관된 값을 여러 `useState`로 쪼개서 상태 동기화가 깨지는 경우 (상태 구조화)
- **Props를 상태로 복사:** 부모로부터 받은 Props를 `useState` 초기값으로 넣었을 때 동기화 안 되는 문제

#### 2. useEffect: 의존성 배열과 클린업의 정석

사이드 이펙트를 제어하지 못하면 무한 루프와 메모리 누수의 지옥을 맛보게 됩니다.

- **무한 렌더링 루프:** 의존성 배열에 넣은 값이 내부에서 계속 업데이트되어 무한 호출되는 케이스
- **클린업 누락:** `setInterval`이나 이벤트 리스너를 등록만 하고 해제하지 않아 메모리가 쌓이는 경우
- **의존성 배열 거짓말:** 함수나 객체를 의존성에 넣었을 때, 매 렌더링마다 참조값이 변해 계속 실행되는 문제
- **Race Condition:** API 호출 결과가 순서대로 도착하지 않아 이전 요청 결과가 화면을 덮어씌우는 현상
- **Effect 내의 과도한 로직:** 하나의 `useEffect`에 너무 많은 일을 시켜서 유지보수가 불가능해진 경우

#### 3. useRef & DOM 제어: 렌더링과 독립된 데이터 관리

리액트의 선언적 방식과 명령적 방식 사이의 균형을 잡는 수련입니다.

- **불필요한 리렌더링 방지:** 값은 변해야 하지만 화면은 다시 그릴 필요 없는 데이터(예: 타이머 ID) 관리
- **DOM 접근 타이밍:** `useRef`가 `null`인 상태에서 접근하여 에러가 발생하는 '마운트 시점'의 이해
- **명령적 API 호출:** 외부 라이브러리(차트, 지도 등)나 HTML5 비디오 태그 등을 직접 제어하는 케이스
- **ForwardRef의 활용:** 커스텀 컴포넌트 내부의 DOM 요소를 부모에게 노출시켜야 하는 정공법 상황
- **변수 유지:** 함수형 컴포넌트 내부에서 일반 변수(`let`)를 썼을 때 렌더링 시 초기화되는 문제 해결

#### 4. Custom Hooks & 로직 분리: 비즈니스 로직의 추상화

UI와 로직을 분리하여 '읽기 좋은 코드'를 만드는 고수의 영역입니다.

- **데이터 페칭 로직:** 여러 컴포넌트에서 중복되는 API 호출 및 로딩/에러 상태 관리 로직 분리
- **폼 핸들링:** 입력값 검증, 제출, 초기화 로직이 수백 줄이 된 컴포넌트를 구원하는 수련
- **브라우저 API 추상화:** LocalStorage, 다크모드, 윈도우 크기 감지 등을 재사용 가능한 훅으로 전환
- **이벤트 핸들러 집합:** 복잡한 마우스/키보드 인터랙션 로직을 하나의 인터페이스로 캡슐화
- **공통 UI 상태:** 모달 오픈/클로즈, 탭 전환 등 단순하지만 반복되는 UI 로직 최적화

#### 5. Context API & 전역 상태: Prop Drilling 탈출

데이터를 전달하는 '고속도로'를 뚫으면서도 성능을 놓치지 않는 수련입니다.

- **불필요한 전체 렌더링:** Context 값이 변할 때 하위 모든 컴포넌트가 다시 그려지는 성능 저하 해결
- **Provider 배치 전략:** 전역 상태가 필요 없는 컴포넌트까지 Provider로 감싸서 생기는 구조적 비효율
- **상태 관심사 분리:** 테마, 유저 정보, 알림 등 성격이 다른 상태를 하나의 거대한 Context로 합친 경우
- **초기값 Undefined 처리:** Context 외부에서 컴포넌트를 사용할 때 발생하는 런타임 에러 방지 정공법
- **Custom Hook과 결합:** `useContext(MyContext)`를 직접 쓰지 않고 `useMyStore()`로 감싸서 사용하는 패턴
